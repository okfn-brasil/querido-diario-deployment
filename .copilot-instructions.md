# GitHub Copilot Guidelines - Querido Diário Deployment

## Arquitetura do Projeto
- **Orquestração**: Kubernetes com Kustomize (base + overlays dev/production)
- **Banco de dados**: CloudNativePG (CNPG) para PostgreSQL
- **Reverse Proxy / Ingress**: Traefik v3 com IngressRoute CRDs
- **TLS**: cert-manager com Let's Encrypt (ClusterIssuer) em produção
- **Storage local (dev)**: Garage (S3-compatible) via Helm
- **Storage (produção)**: S3 externo para arquivos; Ceph RGW para backup do Postgres
- **Cache / Fila**: Redis
- **Busca**: OpenSearch (StatefulSet single-node dentro do cluster k8s, em dev e prod — ver ADR-008)
- **Raspadores**: CronJob Kubernetes (data-processing); execução local via Makefile
- **Automação**: Makefile para comandos k8s, builds Docker e spider setup
- **Ambiente local**: kind (Kubernetes in Docker) com setup/teardown via scripts

## Padrões Kustomize

### Base × Overlay
```yaml
# base/: definições canônicas sem valores de ambiente
# overlays/dev/: patches para ambiente local (kind)
# overlays/production/: patches de produção sem segredos
# overlays/production-local/: GITIGNORED — endpoint Ceph, ClusterIssuer real, etc.
```

### JSON Patch (formato preferido nos overlays)
```yaml
patches:
  - patch: |-
      - op: replace
        path: /spec/storage/size
        value: 100Gi
      - op: add
        path: /spec/backup
        value:
          barmanObjectStore: ...
    target:
      kind: Cluster
      name: postgres
```

### Patch de arquivo separado (para Secrets)
```yaml
patches:
  - path: infra/postgres-credentials-dev.yaml
    target:
      kind: Secret
      name: postgres-credentials
```

## Padrões Traefik IngressRoute

```yaml
apiVersion: traefik.io/v1alpha1
kind: IngressRoute
spec:
  entryPoints:
    - websecure
  routes:
    - match: "Host(`api.queridodiario.org.br`)"
      kind: Rule
      middlewares:
        - name: cors-headers
        - name: security-headers
        - name: compression
      services:
        - name: api
          port: 8080
  tls:
    secretName: queridodiario-tls   # cert-manager cria este Secret
```

- Em dev: `entryPoints: [web]`, sem `tls`, domínio `*.queridodiario.local`
- Em produção: sempre `websecure` + `tls.secretName`; nunca `tls.certResolver`
- Redirecionamentos HTTP→HTTPS via IngressRoute separado (entryPoint `web`)

## Middlewares Traefik (base/traefik-middlewares.yaml)
- `cors-headers` — CORS para API
- `api-rate-limit` — rate limiting da API
- `security-headers` — headers de segurança (HSTS, CSP, etc.)
- `compression` — gzip
- `api-strip-prefix` — remove prefixo `/api` no roteamento por path
- `redirect-to-www` — redireciona apex → www (produção)

## CloudNativePG (CNPG)

```yaml
# base/postgres/cluster.yaml — Cluster CNPG
# overlays/dev: storage 5Gi, limites de memória reduzidos, backup p/ Garage local
# overlays/production: 3 instâncias, 100Gi, storageClass ceph-block-hdd, backup p/ S3
```

- Credenciais via Secret `postgres-credentials` (campo `username` e `password`)
- Backup: `barmanObjectStore` + `ScheduledBackup` CRD (apenas produção)
- Secret de credenciais de backup: `postgres-backup-secret` (keys: `ACCESS_KEY_ID`, `SECRET_ACCESS_KEY`)

## Makefile — Comandos Principais

```bash
# Kubernetes
make k8s-local-up          # Cria cluster kind + sobe ambiente dev
make k8s-local-down        # Destroi cluster kind
make k8s-local-status      # Status dos pods
make k8s-build-dev         # Dry-run overlay dev
make k8s-build-prod        # Dry-run overlay produção

make k8s-apply-prod        # Aplica overlay produção

```

## Convenções de Nomenclatura

### Recursos Kubernetes
- **Namespace**: `querido-diario`
- **Nomes de resources**: kebab-case simples (`api`, `backend`, `celery-worker`)
- **Imagens**: `ghcr.io/okfn-brasil/querido-diario-{service}`
- **Tags dev**: `local`; **tags produção**: versão fixa (ex: `v1.2.3`), nunca `latest`
- **StorageClass produção**: `ceph-block-hdd` (padrão Ceph do cluster)

### Secrets
- `postgres-credentials` — usuário/senha PostgreSQL (CNPG bootstrap)
- `postgres-backup-secret` — credenciais S3 para backup CNPG
- `app-secret` — variáveis sensíveis da aplicação
- `queridodiario-tls` — certificado TLS (criado pelo cert-manager)

## Ambientes

### Desenvolvimento (overlay: `overlays/dev/`)
- Cluster kind local criado por `scripts/k8s_local_up.py` (via `make k8s-local-up`)
- HTTP sem TLS, domínio `*.queridodiario.local`
- Serviços de infra locais: Garage (S3), OpenSearch (StatefulSet, plugin de segurança desabilitado)
- CronJob `data-processing` suspenso — executar manualmente via `make k8s-local-data-processing`
- Frontend build local via `make k8s-local-frontend-build`

### Produção (overlays: `overlays/production/` + `overlays/production-local/`)
- Cluster K3s gerenciado (Revoada/CCSL-IME-USP)
- HTTPS obrigatório; TLS via cert-manager + Let's Encrypt
- Traefik já instalado no cluster (sem ACME próprio — usar `tls.secretName`)
- StorageClass: `ceph-block-hdd` para volumes persistentes
- Backup Postgres: Ceph RGW (S3-compatible) via `barmanObjectStore`
- OpenSearch: StatefulSet single-node dentro do cluster (ver ADR-008), plugin de segurança habilitado
- Valores sensíveis (endpoint Ceph, nome real do ClusterIssuer) em `production-local/` (gitignored)
- Template público: `overlays/production-local.example/`

## Segredos e Privacidade

- **NUNCA commitar** valores reais de credenciais, endpoints internos ou IPs do cluster de produção
- Overlay `production-local/` está no `.gitignore` — use para valores reais
- Valores públicos em `production/` devem usar `<PLACEHOLDER>` quando não puderem ser versionados
- Ver `k8s/overlays/production/DEPLOY.md` para guia completo de deployment

## NÃO FAZER
- Não usar `latest` em tags de imagem de produção
- Não usar `tls.certResolver` nas IngressRoutes (Traefik do cluster não tem ACME configurado)
- Não commitar arquivos `production-local/` ou qualquer credencial real
- Não misturar recursos de infra exclusivos de dev (Garage) no overlay de produção — OpenSearch é a exceção: roda em ambos, mas com manifestos e configuração de segurança distintos por overlay
- Não criar patches de `op: add` sem verificar se o campo já existe na base (usar `op: replace` se existir)
